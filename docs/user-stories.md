# 📋 User Stories – MaintCloud AI

---

## 🧠 Übersicht

Die User Stories beschreiben die Anforderungen aus Sicht der Nutzer und dienen als Grundlage für die Entwicklung sowie Priorisierung.

---

# ✅ Muss-Anforderungen (MVP)

---

### 👨‍🔧 Techniker

- [5 SP] Als Techniker möchte ich Maschinen anlegen können, damit ich sie überwachen kann.

- [5 SP] Als Techniker möchte ich Maschinendaten (z. B. Temperatur, Laufzeit) sehen, damit ich den Zustand beurteilen kann.

- [8 SP] Als Techniker möchte ich den Status (OK, Wartung, kritisch) sehen, damit ich schnell reagieren kann.

- [5 SP] Als Techniker möchte ich Wartungseinträge hinzufügen können, damit ich Maßnahmen dokumentieren kann.

- [3 SP] Als Techniker möchte ich eine Übersicht aller Maschinen sehen, damit ich den Gesamtstatus erkenne.

---

### 🏢 Unternehmen

- [8 SP] Als Unternehmen möchte ich Wartungsbedarfe frühzeitig erkennen, um Ausfälle zu vermeiden.

- [5 SP] Als Unternehmen möchte ich eine zentrale Übersicht über Maschinen haben, um Prozesse zu optimieren.

---

### 👨‍💻 Entwickler / Admin

- [5 SP] Als Entwickler möchte ich eine API bereitstellen, damit das System modular erweitert werden kann.

- [3 SP] Als Admin möchte ich Maschinen bearbeiten und löschen können, damit Daten aktuell bleiben.

---

# 📌 Soll-Anforderungen

---

- [5 SP] Als Nutzer möchte ich Verlaufsdaten sehen, damit ich Trends erkennen kann.

- [8 SP] Als Nutzer möchte ich Cloud-Zugriff, damit ich standortunabhängig arbeiten kann.

- [5 SP] Als Nutzer möchte ich eine einfache Benutzeroberfläche, damit ich das System intuitiv bedienen kann.

- [8 SP] Als Admin möchte ich Benutzer verwalten können, damit der Zugriff kontrolliert ist.

---

# 💡 Kann-Anforderungen

---

- [8 SP] Als Nutzer möchte ich Benachrichtigungen erhalten, wenn ein kritischer Zustand eintritt.

- [13 SP] Als System möchte ich Wartung vorhersagen (Predictive Maintenance).

- [5 SP] Als Nutzer möchte ich Diagramme sehen, um Daten besser zu verstehen.

- [8 SP] Als Nutzer möchte ich das System mobil nutzen können.

---

# 🎯 Priorisierung

1. **Muss (MVP)** → direkt umsetzen  
2. **Soll** → Erweiterung  
3. **Kann** → Zukunft  

---

# 💡 Story Points Erklärung

- **3 SP** → kleine Aufgabe  
- **5 SP** → mittlerer Aufwand  
- **8 SP** → komplex  
- **13 SP** → sehr komplex  

---

# 🚀 Ziel der User Stories

- Anforderungen aus Nutzersicht darstellen  
- Entwicklungsaufwand einschätzen  
- Prioritäten festlegen  
- Grundlage für Kanban & ToDo-Liste schaffen
